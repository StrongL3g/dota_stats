# 🎮 Dota 2 Stats Harvester

> Инструмент для автоматизированного сбора и анализа детальной статистики матчей Dota 2 из личной истории игрока (Steam GCPD).

[![Python](https://img.shields.io/badge/Python-3.8+-blue.svg)](https://python.org)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

---

## 🔍 О проекте

Стандартные API Dota 2 и OpenDota не предоставляют доступ к истории матчей, если игрок скрыл свою статистику в настройках приватности. **Dota 2 Stats Harvester** обходит это ограничение, используя гибридный подход:

1. **JavaScript-скрипт** выполняется в браузере на странице Steam GCPD и извлекает идентификаторы матчей напрямую из локального хранилища.
2. **Python-оркестратор** использует библиотеку `dota2` для подключения к координатору игры и получения детальной статистики: герои, K/D/A, результат матча и другие параметры.

> 💡 Идеально подходит для личного анализа прогресса, подготовки отчётов или интеграции с другими инструментами аналитики.

---

## ⚙️ Требования

- Python 3.8 или выше
- Браузер с доступом к Steam (Chrome, Firefox, Edge)
- Аккаунт Steam с включённой двухфакторной аутентификацией *(рекомендуется)*
- Доступ к странице: `https://steamcommunity.com/id/Legid/gcpd/570?category=Matches&tab=PlayerMatchHistory`

1. Создайте файл `.env` в корневой директории проекта:

## 🚀 Использование

### Шаг 1: Сбор ID матчей (JS-скрипт)

1. Авторизуйтесь в Steam и откройте страницу истории матчей:  
   `https://steamcommunity.com/id/Legid/gcpd/570?category=Matches&tab=PlayerMatchHistory`
2. Откройте инструменты разработчика (`F12`) → вкладка **Console**.
3. Вставьте содержимое скрипта из `scripts/extract_matches.js` и нажмите `Enter`.
4. Дождитесь завершения сбора данных.
5. Введите `stop()` для остановки скрипта.
6. Сохраните вывод в файл `dota_ids_for_api.csv`.

> 📝 Скрипт автоматически обрабатывает пагинацию и извлекает все доступные матчи.

### Шаг 2: Загрузка детальной статистики (Python)

Запустите основной скрипт:
```bash
python run_analytics.py
```

После завершения в файле `analytics_report.csv` появится отчёт со статистикой.

## 📊 Формат отчёта

Файл `analytics_report.csv` содержит следующие колонки:

| Match ID | Hero | K / D / A | Win | Duration | Date |
|----------|------|-----------|-----|----------|------|
| 8767848614 | Jakiro | 5 / 2 / 18 | Yes | 42:15 | 2024-01-15 |
| 8767773235 | Axe | 12 / 8 / 5 | No | 38:42 | 2024-01-14 |

> 📌 Формат можно расширить, добавив новые поля в `run_analytics.py`.

## 🔐 Безопасность

- 🔒 Никогда не публикуйте файл `.env` с учётными данными.
- 🔒 Используйте отдельный Steam-аккаунт для тестирования, если возможно.
- 🔒 Скрипт не сохраняет сессионные токены — аутентификация выполняется «на лету».
- ⚠️ Использование инструмента должно соответствовать [Условиям использования Steam](https://store.steampowered.com/subscriber_agreement/).

## 🛠 Устранение неполадок

| Проблема | Возможное решение |
|----------|------------------|
| `Authentication failed` | Проверьте логин/пароль, убедитесь, что Steam Guard отключён или используйте Shared Secret |
| `No matches found` | Убедитесь, что страница GCPD загружена полностью, и у аккаунта есть история матчей |
| `ModuleNotFoundError` | Установите зависимости: `pip install -r requirements.txt` |
| JS-скрипт не работает | Попробуйте другой браузер или обновите страницу GCPD перед запуском |
